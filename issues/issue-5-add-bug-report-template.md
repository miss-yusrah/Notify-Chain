## Description
When users find bugs, it's helpful to have a standardized format for them to report issues so maintainers get all the necessary information (OS, rust version, steps to reproduce). GitHub allows creating issue templates for this purpose.

## Task
1. Create a `.github/ISSUE_TEMPLATE/bug_report.md` file.
2. Add sections for:
   - Description of the bug
   - Steps to reproduce
   - Expected behavior
   - Screenshots/Logs
   - Environment (OS, Rust version, Soroban CLI version)

## Expected Outcome
A pull request containing the new bug report template file formatted cleanly in Markdown.
