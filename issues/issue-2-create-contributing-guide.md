## Description
To help new contributors get started with our project quickly, we need a `CONTRIBUTING.md` file in the root directory. This guide will outline the steps to set up the project locally, run tests, and submit pull requests.

## Task
1. Create a `CONTRIBUTING.md` file in the root directory.
2. Add a brief introduction welcoming contributors.
3. Add a section detailing how to clone the repo, install dependencies (`cargo`, Soroban CLI), and run the tests.
4. Add a section explaining the pull request process.

## Expected Outcome
A pull request adding `CONTRIBUTING.md` with clear and helpful instructions for future developers.
